# Phase-Based Development Plan

## Phase 1 — Minimum Working System

Implemented in this package.

- Docker Compose
- PostgreSQL schema
- Backend API
- Web dashboard
- Detector log import
- Signal log import
- Video file registration
- Basic charts and tables

## Phase 2 — Data Normalization and Historical Analytics

Planned additions:

- Detector-to-lane mapping screen
- Approach direction labels
- Data validation dashboard
- Missing 15-minute interval report
- Daily, hourly, AM peak, PM peak charts
- CSV export

## Phase 3 — Forecasting and Signal Recommendation

Planned additions:

- Historical average forecast
- Moving average forecast
- Gradient boosting forecast
- MAE, RMSE, MAPE evaluation
- Rule-based green-time recommendation
- Recommendation explanation log

## Phase 4 — Video Processing and Vehicle Detection

Planned additions:

- Frame extraction jobs
- YOLO/ONNX vehicle detection
- Detection metadata table
- Detection overlay preview
- RTSP source configuration

## Phase 5 — Tracking and Incident Detection

Planned additions:

- Object tracking
- Stalled vehicle detection
- Queue-zone occupancy
- Queue spillback alert
- Snapshot saving
- Event review screen

## Phase 6 — Final Packaging

Planned additions:

- Test plan and validation report
- User manual
- Admin manual
- Training material
- Open-source component list
- Completion report template
